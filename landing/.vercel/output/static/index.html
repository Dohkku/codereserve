The Reality

Maintainers are burnt out. Your repo is getting flooded with low-quality contributions from new accounts and automated tools. Filtering noise is unpaid labor. If this continues, maintainers will quit and critical infrastructure will rot. You shouldn't have to choose between an open repo and your sanity.

The Mechanism

We introduce a Reputation Shield for Pull Requests. Phase 1 - The Gatekeeper: When a PR is opened, we calculate a Risk Score based on GitHub account age, historical merges in protected repos, and whitelist status. Phase 2 - The Outcome: • Low Risk: PR accepted immediately. Label: CR-Trusted. • High Risk (new/flagged accounts): PR auto-closed with instructions. Phase 3 - The Security Deposit (Bail): Unverified users can deposit $5 to re-open their PR. • PR Merged or Closed: Deposit refunded automatically. • Maintainer flags as Spam: Deposit slashed to repo treasury. Result: Trusted devs work freely. New contributors prove intent with a refundable deposit. Spam becomes financially painful. --- FOR MAINTAINERS Stop reviewing noise. FAQ

Don't similar tools already exist? (Tea.xyz, Gitcoin...)

Yes. Their incentives are broken. Tea.xyz: Pays based on dependency graphs. This incentivizes low-quality submissions to farm tokens. They add noise; we filter it. Gitcoin: Relies on donations and goodwill. We're a merit-based system: proven contributors work freely, unknowns prove intent with a refundable deposit. We're Reputation First. The deposit is a last resort, not a gate.

Why Blockchain?

Programmable escrow, transparent treasuries, and permissionless reputation. Traditional payment rails don't support automatic refunds based on PR outcomes. We use crypto as infrastructure. No tokens. No speculation.

Is this pay-to-play?

No. It's Reputation First. Trusted contributors (good account history, previous merges) work for free. Only unverified accounts need to provide a Security Deposit, and it's fully refunded when the PR is reviewed. If you're legitimate, it costs nothing.

Who runs this?

One dev building the MVP. Goal: decentralized protocol governed by the maintainers and contributors doing the work.
